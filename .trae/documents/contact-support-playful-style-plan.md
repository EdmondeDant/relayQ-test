# 联系页俏皮化改版计划

## Summary
- 目标：在**不改逻辑代码**的前提下，将 `frontend/src/views/user/ContactSupportView.vue` 中的 `section`、`article`、`aside` 三个核心区域从偏正式的企业风格，调整为更轻松的“手账贴纸感”。
- 额外需求：在页面中加入联系电话 `+8617333910159`，并明确说明“上午不要找他，你也找不到，他是个技术宅，从下午到半夜处理业务”。
- 范围：仅调整该页面的模板结构、文案语气和 Tailwind 类名；不改接口、不改状态流、不改发布群二维码逻辑。

## Current State Analysis
- 文件位置：`frontend/src/views/user/ContactSupportView.vue`
- 当前实现特点：
  - 顶部 `section` 使用白底 + 深色描边 + 阴影，语气偏正式。
  - 二维码展示区 `article` 信息层次清晰，但视觉语言偏“后台卡片”，缺少轻松感。
  - 右侧 `aside` 仅展示微信、QQ 和可选 `contactInfo`，没有电话信息，也没有联系时间偏好说明。
  - 联系方式和提示文案多为功能性描述，例如“联系站长，提出你的想法和建议。一起交流。”、“等待管理员发布”等，整体比较正式。
- 当前逻辑边界：
  - 微信、QQ 是本地常量：`wechatId`、`qqId`
  - 电话目前未出现在页面中
  - 群二维码读取/发布/清空逻辑已独立稳定，应避免改动 `loadContactPage()`、`publishQrCode()`、`clearQrCode()`、`copyContact()` 等逻辑函数

## Proposed Changes

### 1. 调整顶部 `section` 为“手账贴纸感”欢迎区
- 文件：`frontend/src/views/user/ContactSupportView.vue`
- 变更内容：
  - 保留现有结构，但将正式白卡视觉改为更柔和、更有贴纸层次的组合，例如：
    - 更轻的渐变底色
    - 更圆润的边框和阴影
    - 小标签/贴纸式辅助文案
  - 标题和副标题改成更随和的表达，减少“官方公告”感。
  - 微信 / QQ 两个联系卡改成更像手账标签卡片的样式，按钮也同步变得更轻松，不再是纯黑色商务按钮。
- 目的：
  - 让用户一进页面就感到“这是能直接聊、能直接问”的页面，而不是标准后台页。

### 2. 调整二维码 `article` 的状态表达和空态语气
- 文件：`frontend/src/views/user/ContactSupportView.vue`
- 变更内容：
  - 保留 `displayedQr`、`isAdmin`、`publishedQr` 等现有判断，不改条件逻辑。
  - 将“已发布 / 未发布”“扫码进群 / 等待管理员发布”的视觉表达改为更轻松的贴纸/状态签样式。
  - 优化空态文案，使其更口语化、更像真人沟通，而不是系统提示。
  - 管理员说明区保持原行为，但文案改为更轻松、更友好的提示语。
- 目的：
  - 在不改发布逻辑的前提下，让状态表现更自然，不那么“正式后台”。

### 3. 扩展右侧 `aside`，新增联系电话和联系时间说明
- 文件：`frontend/src/views/user/ContactSupportView.vue`
- 变更内容：
  - 在现有微信号、QQ 号卡片下方新增“联系电话”卡片：
    - 展示号码：`+8617333910159`
    - 可直接用现有 `copyContact()` 交互添加“复制电话”按钮，或在计划执行时仅保留静态展示，优先选择不增加额外逻辑负担的方案
  - 在电话卡片或其下方增加明确说明：
    - “上午不要找他，你也找不到。”
    - “他是个技术宅，通常从下午到半夜处理业务。”
  - 整体 `aside` 卡片视觉统一改成手账贴纸/便签板风格，和顶部、二维码区保持同一套语气。
- 目的：
  - 把电话信息作为真实有效的联系入口露出，同时通过文案提前管理用户预期。

### 4. 统一三块区域的文案语气与视觉细节
- 文件：`frontend/src/views/user/ContactSupportView.vue`
- 变更内容：
  - 统一 `section`、`article`、`aside` 的色彩、圆角、边框、阴影、标签小字风格。
  - 将“联系我们”“联系信息”“官方群二维码”“管理员发布区”等标题保留原语义，但语气更轻松。
  - 适度加入贴纸式小标签、轻提示、小注释，形成“手账感”，但避免影响可读性和信息层级。
- 目的：
  - 页面整体风格一致，避免出现顶部活泼、右侧仍旧正式的割裂感。

## Assumptions & Decisions
- 决策：采用“手账贴纸感”作为主视觉方向，而不是糖果卡片或纯聊天气泡风。
- 决策：联系电话主要放在右侧 `aside` 的联系信息区，而不是顶部主视觉区。
- 决策：不修改 API、store、后端、路由或页面行为逻辑；仅做模板与样式层调整。
- 决策：优先复用当前页面已有的 Tailwind 工具类体系，不引入新的组件或样式文件。
- 假设：联系电话允许直接公开展示在该页面中，且备注文案可按用户原意做适度润色，但不改变含义。

## Verification Steps
- 视觉核对：
  - 打开 `/contact-support` 页面，确认 `section`、`article`、`aside` 三块整体风格明显比当前更轻松、更俏皮。
  - 确认三块区域样式统一，没有出现局部风格跳脱。
- 内容核对：
  - 确认页面展示联系电话 `+8617333910159`
  - 确认电话旁或下方有“上午不要找他，你也找不到；从下午到半夜处理业务”的明确说明
- 行为回归：
  - 管理员态下，上传二维码、发布按钮、清空按钮仍按原有逻辑工作
  - 非管理员态下，二维码展示与空态提示仍正常
  - 微信 / QQ 复制按钮不受影响；如果新增电话复制按钮，则一并验证复制成功提示
- 风险检查：
  - 不出现布局溢出、深浅色模式可读性下降、按钮状态不清晰等问题
