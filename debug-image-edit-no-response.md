[OPEN] image-edit-no-response

## 症状
- 在线体验里的“图片编辑”已选择图片、API Key 和模型。
- 点击执行后页面显示“正在处理真实任务...”，但看起来没有结果返回，用户感知为“没有反应”。

## 期望
- 图片编辑应像 AI 生图一样，发出真实请求并返回编辑结果；如果失败，应显示明确错误。

## 当前可证伪假设
1. 图片编辑前端没有真正发出编辑请求，UI 只是进入了 loading 状态。
2. 前端发出了请求，但调用了错误的接口或漏传了图片文件字段。
3. 后端图片编辑请求走到了 `/v1/images/edits`，但账号调度或分组筛选失败，没有把错误反馈到页面。
4. 上游模型 `gpt-image-2-adobe` 不支持当前实现的图片编辑参数格式，导致请求悬挂或静默失败。
5. 图片编辑成功了，但前端结果解析或资产落盘链路没有把结果显示出来。

## 调试计划
1. 定位前端“图片编辑”入口与请求参数构造。
2. 对比 AI 生图成功链路和图片编辑链路的接口差异。
3. 加最小化埋点，记录图片编辑请求是否发出、是否选到账号、是否返回结果。
4. 复现并基于日志判断根因，再做最小修复。
