## 十二、7 月 14 日补充更新（二）— 一键安装国内网络中转与自动回退

### 1. 背景问题

- 很多客户在使用项目首页的一键安装命令安装 `OpenClaw` / `Hermes` 时，会在下载 `Node.js`、`Git`，以及 Hermes 官方安装脚本和仓库快照过程中，因为国内网络访问上游站点不稳定而失败。
- 用户诉求是：
  - 不要求客户手动替换源
  - 不要求客户修改命令
  - 继续保持一条命令自动安装

### 2. 本次改造目标

- 将 Windows 一键安装链路改造成：
  - 优先走 RelayQ 自己站点的 `/downloads/...` 中转地址
  - 中转失败时自动回退到官方源
- 保证客户端用户仍然只执行原有一键安装命令，不需要任何手工干预。

### 3. OpenClaw Windows 安装链路增强

- 在 `OpenClaw` Windows 安装脚本里新增依赖预检查：
  - 自动检测 `Node.js`
  - 自动检测 `Git`
- 若缺失则自动下载安装：
  - 先尝试 RelayQ 自己站点中转地址
  - 失败后回退到官方地址
- 依赖就绪后，再继续执行 OpenClaw 官方安装脚本和原有 onboarding 流程。

### 4. Hermes Windows 安装链路增强

- 在 `Hermes` Windows 安装脚本里新增依赖预检查：
  - 自动检测 `Node.js`
  - 自动检测 `Git`
